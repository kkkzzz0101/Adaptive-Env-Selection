\begin{frame}{Background: GRPO Signal Is State-Dependent}
  \begin{itemize}
    \item GRPO uses group-relative advantages from sampled rollouts.
    \item Too easy: most rollouts are correct, so the relative update signal is small.
    \item Too hard: most rollouts fail, so the relative update signal is also small.
    \item The useful training distribution changes as the model learns.
  \end{itemize}
\end{frame}

\begin{frame}{Formulation: Non-stationary Bandit}
  \begin{itemize}
    \item We model each curriculum cluster as a bandit arm.
    \item Arm value is estimated from recent absolute advantage.
    \item Non-stationarity: the best arm changes during training.
    \item Goal: adapt sampling while keeping the curriculum structure interpretable.
  \end{itemize}
\end{frame}

\begin{frame}{Starting Point: DUMP}
  \begin{itemize}
    \item DUMP tracks distribution-level absolute advantage with a sliding window.
    \item It uses UCB to sample distributions with high estimated learnability or uncertainty.
    \item We keep the core idea: advantage magnitude is a learner-state signal.
    \item DURCL changes the state tracker and adds micro-bucket rebucketing.
  \end{itemize}
\end{frame}

\begin{frame}{Why Mixed Math+Zebra Needs More}
  \begin{itemize}
    \item Different scoring standards and output formats make raw task signals less directly comparable.
    \item Math and Zebra have different problem geometry and learning rates.
    \item A nominal difficulty cluster may not stay internally coherent after training starts.
    \item Distribution-level scheduling changes weights, but does not repair bad cluster membership.
  \end{itemize}
\end{frame}

\begin{frame}{Scheduler Core}
  \begin{itemize}
    \item Replace DUMP's sliding window with exponential-decay advantage state:
    \[
      s_i \leftarrow \lambda s_i + (1-\lambda)|A_i|.
    \]
    \item Cluster sampling by UCB plus softmax:
    \[
      U_c = v_c + \beta \sqrt{\frac{2\log(N+1)}{n_c+1}}.
    \]
    \item Rebucketing is local: compare each micro-bucket only with its current and adjacent clusters.
  \end{itemize}
\end{frame}

\begin{frame}{Micro-bucket Rebucketing}
  \begin{itemize}
    \item Split each task--cluster group into micro-buckets; individual samples do not move alone.
    \item Track each micro-bucket by dynamics signature:
    \[
      z_g=(\ell_g,s_g),
    \]
    where $\ell_g$ is recent mean $|\hat A|$ and $s_g$ is the fitted slope.
    \item Compare to current-or-adjacent cluster centers:
    \[
      j^\star(g)=\arg\min_{j\in\{k-1,k,k+1\}}\|\tilde z_g-\tilde z_j\|_2^2.
    \]
    \item Apply adjacent mutual swaps only after consistent checks.
  \end{itemize}
\end{frame}

\begin{frame}{Limitations}
  \begin{itemize}
    \item Current comparisons are limited-run and do not include confidence intervals.
    \item Difficulty-only initialization may not match model competence.
    \item Rebucketing changes task composition, not only difficulty.
    \item Accuracy-initialized rebucketing result is still pending.
  \end{itemize}
\end{frame}

\begin{frame}{Conclusion and Next Step}
  \begin{itemize}
    \item DURCL models curriculum scheduling as a non-stationary bandit.
    \item Micro-bucket rebucketing corrects local cluster mismatch with adjacent mutual swaps.
    \item Current evidence: UCB drift, step-200 gains, and interpretable composition correction.
    \item Main next step: finish acc-init rebucketing and run multi-seed evaluation.
  \end{itemize}
\end{frame}
